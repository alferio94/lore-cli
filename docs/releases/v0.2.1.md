# v0.2.1

## Highlights
- Fixes the Windows installer so the documented `install.ps1` flow works on Windows PowerShell 5.1 as well as PowerShell 7+ on Windows.
- Keeps PATH updates explicit: `install.sh --add-to-path` and `install.ps1 -AddToPath` are still opt-in, and default installs do not prompt or mutate PATH.
- Clarifies post-install guidance: installers print the direct binary path immediately, and PATH opt-in users may need a new terminal/session before `lore` is available by name.

## Validation Notes
- Focused Unix installer smoke checks were run locally.
- Windows PowerShell 5.1 installer smoke coverage is validated locally on a real Windows machine, not in the release workflow.
- This note intentionally avoids claiming broader installer automation than those focused checks provide.
