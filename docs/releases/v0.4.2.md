# v0.4.2

## Highlights
- Adds Codex as a supported config-only install target.
- Adds a durable `agent-config.json` contract for canonical SDD agent model defaults.
- Updates Pi install defaults to use hosted Lore MCP via `pi-mcp-adapter`.

## Codex config-only install
- `lore install --target codex` now projects Lore-managed configuration into `~/.codex`.
- Existing `~/.codex/agents.md` is backed up before being overwritten with clean Lore-managed instructions.
- Codex receives managed skills and manifest-backed install state.
- Codex projection consumes `agent-config.json`, so custom SDD agent model values are reflected in generated Codex-facing config.
- Scope remains config-only: no Lore MCP config, no `config.toml` MCP block, no `codex exec` runner, no live subagent invocation, and no npm bootstrap.

## Agent config contract
- Adds `agent-config.json` beside auth-owned `config.json` under the Lore config directory.
- Declares canonical SDD phase agents with initial model `gpt-5.4`.
- Keeps login/logout auth rewrites isolated from agent configuration.
- Status, doctor, and install diagnostics inspect the contract without executing agents.

## Pi hosted MCP install
- Default Pi install uses hosted Lore MCP through `pi-mcp-adapter`.
- The installer manages Pi `settings.json` and `mcp.json` for the hosted `/v1/mcp` endpoint.
- Legacy `lore-memory` Pi extension assets remain available for explicit rollback/testing but are not installed by default.
