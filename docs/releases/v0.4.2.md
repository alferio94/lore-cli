# v0.4.2

## Highlights
- Adds Codex as a supported install target with managed remote Lore MCP config.
- Adds a durable `agent-config.json` contract for canonical SDD agent model defaults.
- Updates Pi install defaults to use hosted Lore MCP via `pi-mcp-adapter`.

## Codex managed remote MCP install
- `lore install --target codex` now projects Lore-managed configuration into `~/.codex`.
- Existing canonical `~/.codex/AGENTS.md` is backed up before being overwritten with clean Lore-managed instructions.
- Codex receives managed skills, manifest-backed install state, and a managed `~/.codex/config.toml` Lore MCP entry.
- The Codex MCP config points to `${lore_url}/v1/mcp` and writes `Authorization = Bearer <saved-token>` from the existing Lore login/keychain flow.
- Codex projection consumes `agent-config.json`, so custom SDD agent model values are reflected in generated Codex-facing config.
- Codex still does not install Codex itself, run `codex exec`, enable live subagent invocation, or add npm bootstrap behavior.

## Agent config contract
- Adds `agent-config.json` beside auth-owned `config.json` under the Lore config directory.
- Declares canonical SDD phase agents with initial model `gpt-5.4`.
- Keeps login/logout auth rewrites isolated from agent configuration.
- Status, doctor, and install diagnostics inspect the contract without executing agents.

## Pi hosted MCP install
- Default Pi install uses hosted Lore MCP through `pi-mcp-adapter`.
- The installer manages Pi `settings.json` and `mcp.json` for the hosted `/v1/mcp` endpoint.
- Legacy `lore-memory` Pi extension assets were removed from the install bundle and are not available in any install path. The optional `lore-footer` (UI status) remains available for explicit opt-in via `lore install --target pi --component pi-extensions` but is not installed by default.
