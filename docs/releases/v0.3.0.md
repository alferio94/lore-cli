# v0.3.0

## Highlights
- Adds password-first `lore login` with email/password against `POST /v1/auth/login`, followed by `GET /v1/me` validation before saving session state.
- Adds `--password-stdin` for non-interactive password input without exposing secrets in argv.
- Stores the reusable Bearer token in the OS keychain while keeping only non-secret login metadata in local config.
- Keeps manual `--token` login working as a compatibility path for older servers and existing automation.
- Updates the TUI login flow to start with email + masked password input while preserving compatibility-token fallback.
- Keeps `lore logout` local-only and idempotent: it removes saved metadata and the matching keychain credential without attempting server-side revocation.

## Upgrade notes
- Password login is now the primary documented path.
- Headless Linux environments still require a working Secret Service/keyring session; Lore fails closed instead of falling back to plaintext token storage.
- Compatibility-token login remains supported with `lore login --server <url> --token <token>`.
